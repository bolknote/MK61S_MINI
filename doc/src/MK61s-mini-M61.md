# Инструкция по файлам M61

Версия инструкции: 22.07.2026

Файл `.m61` - это текстовый сценарий для MK61s mini. Он может просто загрузить
программу МК-61 в программную память, а может выполнить подготовку, запустить
программу, дождаться ее остановки, проверить стек или регистры и открыть другой
сохраненный файл.

В хранилище устройства такие файлы имеют тип `M1`. На USB-диске они видны как
обычные файлы с расширением `.M61` или `.m61`.

## Кратко

- `.m61` - обычный текстовый файл.
- Одна строка файла - одна команда.
- Разрешенные сценарные команды пишутся в том же виде, что и в терминале
  устройства; административные команды в сценариях запрещены.
- Пустые строки игнорируются.
- Строки вида `:label` задают метки для переходов.
- `run` запускает загруженную программу МК-61 и продолжает сценарий после ее
  остановки.
- `run :label` переходит на метку внутри текущего `.m61`.
- `trap <address> run :label` перехватывает шаг программы до его выполнения.
- `ret` возвращает из обработчика, из вложенного `.m61` либо завершает корневой
  сценарий.
- `print "..."` пишет байты и значения регистров на текущий экран без
  неявной очистки и без неявного перевода строки.
- `if ... <command>` выполняет команду только при истинном условии.
- `open <path>` открывает или запускает сохраненный файл и после выхода
  продолжает сценарий.
- Квота общая для всех типов и каталогов и зависит от ёмкости SPI NOR:
  192 узла на 512 КиБ и 4084 узла на штатной 16-МиБ W25Q128.
- Максимальный размер файла в хранилище - 1536 байта.

## Структура файла

Файл состоит из строк:

```text
hin 0000 0144004501000049010C0B024A09044C0108070207030806
hin 0024 0E0103000000000600384101030708070303060E01000006
run
```

В этом примере первые две строки записывают байты в программную память МК-61,
а `run` запускает программу.

Имена команд и их числовые аргументы записываются ASCII-символами. Пути и имена
файлов могут содержать валидный UTF-8. Для имен команд используется нижний
регистр, как в примерах: `hin`, `run`, `open`, `if`.
Специальная команда записи регистра пишется с большой `R`: `R0= ...`.

Поддерживаются переводы строк LF, CRLF и CR. Управляющие символы внутри строки не
нужны. Комментариев в синтаксисе нет: неизвестная команда считается ошибкой и
останавливает сценарий.

Одна строка сценария должна помещаться в командный буфер терминала. Практически
это означает не более 239 символов полезного текста в строке.

## Имена и импорт с USB-диска

При копировании файла на USB-диск устройства расширение определяет тип:

| Расширение | Тип в хранилище | Что открывает |
| --- | --- | --- |
| `.m61` | `M1` | сценарий МК-61 |
| `.foc` | `F1` | FOCAL |
| `.tbi` | `B2` | TinyBASIC |
| `.txt` | `T1` | текстовый просмотрщик |
| `.state.txt` | `M2` | снимок состояния МК-61 |

Basename ограничен 31 байтом валидного UTF-8. Русские и другие Unicode-имена
сохраняются без транслитерации. Слишком длинное, запрещённое FAT-имя либо
регистронезависимый дубликат отклоняются, а не переименовываются молча.

Команды `open` и `run <path>` принимают путь с расширением или без него:

```text
open GAME.m61
open GAME
open "Projects/Мои программы/GAME.m61"
open ../lib/COMMON.m61
```

Поддерживаются абсолютные и относительные пути, `/` и `\`, компоненты `.` и
`..`. Во вложенном сценарии относительный путь считается от папки вызывающего
файла. Для одиночного имени сохранён fallback в корень. Если расширение не
указано и одинаковый basename имеют файлы разных типов, команда возвращает
ошибку ambiguity: нужно дописать расширение.

## Загрузка программы МК-61

Главная команда для загрузки программы из `.m61` - `hin`:

```text
hin <addr> <hex-bytes>
```

`<адрес>` - десятичный адрес из четырех цифр. `<hex-байты>` - слитная строка
шестнадцатеричных байтов. Каждый байт занимает две hex-цифры:

```text
hin 0000 010203040506
hin 0024 0E01030000000006
```

Адреса `0000`...`0104` относятся к классической памяти на 105 шагов. Если
сценарий записывает шаги выше классической области или использует команды
регистра `RF`, устройство автоматически включает расширенную память до 112
шагов.

Когда текущая программа МК-61 сохраняется средствами устройства, она
записывается как `.m61` с набором строк `hin`. Автоматический `run` в конец не
добавляется. Такой файл при открытии только загружает программу; чтобы он сразу
запускался, добавьте строку:

```text
run
```

## Запуск и возврат

Команда без аргументов:

```text
run
```

запускает текущую программу МК-61 так же, как последовательность `F`, `/-/`,
`В/О`, `С/П`. В `.m61` это асинхронная точка ожидания: сценарий не идет дальше,
пока программа МК-61 не остановится. Если программа ждет нажатий с клавиатуры,
сценарий тоже ждет ее остановки.

После остановки программы выполнение продолжается со следующей строки `.m61`:

```text
hin 0000 010203...
poke X 1.25e02
run
open RESULT.txt
```

Команда `ret` зависит от текущего контекста:

- в обработчике `trap` она восстанавливает калькулятор и продолжает программу с
  перехваченного шага;
- во вложенном `.m61` она немедленно возвращает в вызывающий сценарий;
- в корневом `.m61` она завершает сценарий.

Достижение конца обычного корневого или вложенного файла сохраняет прежнее
поведение завершения/возврата. Обработчик ловушки обязан выполнить явный `ret`:
выход из него через конец файла считается ошибкой.

## Ловушки программы МК-61

Ловушка объявляется строкой:

```text
trap <address> run :label
```

Например:

```text
trap 105 run :message

run
ret

:message
print "AT 105: X={X}\r\n"
ret
```

Строка `trap` активирует ловушку при своём выполнении. Это позволяет сначала
запустить подготовительные программы, а затем объявить перехват только для
основной программы. Все объявления и целевые метки всё равно проверяются при
открытии файла: адрес не может повторяться, а отсутствующая метка является
ошибкой до выполнения первой команды.

Адрес — десятичный номер opcode от `0` до `111`. В классическом режиме
доступны шаги `0`...`104`; адреса `105`...`111` требуют расширенной памяти.
Если активная ловушка лежит вне текущей программной памяти, следующий `run`
завершается ошибкой. Для двухбайтовой команды указывается адрес её opcode, а не
адрес байта-аргумента.

При достижении адреса ядро заканчивает предыдущую команду, но ещё не исполняет
перехваченную. Полный контекст калькулятора сохраняется в оперативной памяти:
кольцо, стек, регистры, X2, IP, режимы и внутреннее состояние микросхем. Затем
сценарий переходит на метку. `ret` восстанавливает контекст байт-в-байт и
разрешает ровно одно выполнение той же команды; при следующем приходе на адрес
ловушка снова сработает. При ошибке или отмене сценария сохранённый контекст
также восстанавливается.

Пока контекст сохранён, обработчику доступны только безопасные операции,
которые не двигают и не изменяют калькулятор: `print`, `ret`, `if`,
`run :label`, `ver`, `list`, `dump`, `pub`, `lasm`, `isa`, `hout`, `reg`,
`stk`, `1302`, `ring`, `led` и `beep`. Обычный `run`, `open`, `load`, `kbd`,
`cmd`, `hin`, `asm`, `poke` и запись регистров в обработчике запрещены.
Тот же фильтр временно применяется к параллельному интерактивному терминалу,
чтобы команда с компьютера не изменила ядро между сохранением и `ret`.

Несколько адресов могут вести на одну метку:

```text
trap 31 run :wall
trap 44 run :wall
```

## Вывод `print`

`print` принимает ровно одну строку в двойных кавычках и пишет её в текущий
экранный буфер калькулятора. Команда ничего не добавляет к строке автоматически:

```text
print "TEST"
print "X={X}, R0={R0}, X2={X2}\r\n"
print "\x1B[3;5H*"
print "\x1B[2K\rnew line"
print "\xFF"
```

Без суффикса подстановки имеют фиксированный индикаторный формат калькулятора:

| Подстановка | Значение |
| --- | --- |
| `{X}`, `{Y}`, `{Z}`, `{T}` | стековые регистры |
| `{X1}` | скрытый стековый регистр X1 |
| `{X2}` | отдельный регистр текущей индикации |
| `{R0}`...`{RE}` | регистры памяти |
| `{RF}` | RF в расширенном режиме |
| `{0}`...`{F}` | короткие синонимы `{R0}`...`{RF}` |

Для числового регистра или стека допустим короткий формат после двоеточия:

| Формат | Что выводится |
| --- | --- |
| `{RA:m}` | первая цифра нормализованной мантиссы |
| `{RA:e}` | модуль десятичного порядка без ведущего нуля |

Например, игровая координата `RA = 3e-6` даёт `3` через `{RA:m}` и `6` через
`{RA:e}`. Так сценарий может разобрать принятую в «Полёте шмеля» упаковку
координат и сам уложить поле в доступные строки экрана. Форматы
регистронезависимы (`:M`, `:E` тоже допустимы); к индикаторной строке `X2` они
не применяются.

Имена подстановок регистронезависимы. `{{` и `}}` печатают литеральные фигурные
скобки. Поддерживаются escape-последовательности `\\`, `\"`, `\r`, `\n`,
`\t`, `\b` и `\xHH` с ровно двумя hex-цифрами. `\xHH` передаёт экрану ровно
один байт, поэтому `\xFF` — именно код `FF`, без UTF-8-перекодирования.

ANSI разбирает сама прошивка до записи в активный дисплей — физический или USB
Screen. Поддерживаются перемещения `A/B/C/D/E/F`, горизонтальная позиция `G`,
позиционирование `H/f`, очистка `J/K`, сохранение и восстановление курсора
`s/u`, а также `ESC 7`/`ESC 8`. Координаты ограничиваются фактическими
`cols()` и `rows()` дисплея. SGR `...m` принимается, но цвет и другие атрибуты
на монохромном экране не имеют эффекта.

Обычный текст заменяет только занятые им ячейки. `print` не очищает ни строку,
ни экран; очистка происходит только по явно напечатанным `J` или `K`.

## Метки и переходы

Метка - строка, которая начинается с двоеточия:

```text
:loop
```

Метка сама ничего не выполняет. Переход на метку выполняется командой:

```text
run :loop
```

При открытии файла метки один раз проверяются и индексируются, поэтому переход
не перечитывает весь файл. Переход может быть и вперед, и назад; метка ищется
только в текущем файле, не во вложенном сценарии. Имя метки содержит от 1 до 31
непробельного ASCII-символа, имена должны быть уникальны, максимум — 64 метки.

Пример цикла:

```text
:again
run
if r0>0 run :again
beep 1200,80
```

Здесь программа МК-61 запускается снова, пока регистр `R0` больше нуля.

## Условия

Команда `if` выполняет другую команду, только если условие истинно:

```text
if <left><op><right> <command>
```

Поддерживаемые операции:

| Операция | Значение |
| --- | --- |
| `>` | больше |
| `>=` | больше или равно |
| `<` | меньше |
| `<=` | меньше или равно |
| `==` | равно |
| `!=` | не равно |

Операндами могут быть числа, регистры стека и регистры памяти:

| Операнд | Что читает |
| --- | --- |
| `x` | стековый регистр X |
| `y` | стековый регистр Y |
| `z` | стековый регистр Z |
| `t` | стековый регистр T |
| `x1` | скрытый регистр X1 |
| `r0`...`re` | регистры памяти R0...RE |
| `rf` | регистр RF, если включена расширенная память |
| `1.25e02`, `-5`, `0` | числовой литерал |

После условия можно поставить любую обычную команду сценария, включая переход,
запуск или открытие файла:

```text
if x<0 open NEGATIVE.txt
if r3==10 run :done
if y!=0 run
if rf>=1.5e03 beep 2000,100
:done
```

Если условие ложно, строка ничего не делает и сценарий переходит к следующей
строке.

## Вложенные файлы

`open <path>` открывает сохраненный файл относительно папки текущего сценария:

```text
open HELP.txt
open SETUP.m61
open TEST.foc
open SAMPLE.tbi
open ../lib/COMMON.m61
```

В `.m61` это работает как вызов с возвратом. Просмотрщик текста, FOCAL,
TinyBASIC, шрифт или вложенный `.m61` выполняются, а после выхода управление
возвращается к следующей строке исходного сценария.

`run <path>` - синоним `open <path>`:

```text
run SETUP.m61
```

Для вложенных `.m61` есть стек возвратов на 8 уровней. Если глубина вложенности
превышена или файл не найден, сценарий останавливается с ошибкой.

Команда:

```text
load <slot>
```

в интерактивном терминале загружает программу из числового файла `M1`. В
`.m61` она выполняет сохраненный сценарий `M1` с внутренним именем этого номера
и возвращается назад. Например, `load 03` вызывает сценарий с именем `3`.
Для именованных файлов понятнее использовать `open <path>.m61`.

## Работа со стеком и регистрами

Для записи в стек используется `poke`:

```text
poke X 1.25e02
poke Y -3.14e00
poke Z 0e00
poke T 5e01
```

Имя стекового регистра - `X`, `Y`, `Z` или `T`. Значение задается обычным
десятичным числом, например `3.14`, `-5` или `1.25e02`. Допустимый порядок
результата — от `-99` до `99`.

Регистры памяти можно читать в условиях через `r0`...`re` и `rf`.
Запись в регистр памяти выполняется специальной командой:

```text
R0= <value>
RA= <value>
RE= <value>
RF= <value>
```

`RF` доступен только в расширенном режиме памяти. `R<r>=` принимает такое же
проверяемое десятичное число, как `poke`; напрямую испортить внутренние BCD-тетрады
регистра через терминал нельзя.

## Клавиатура и коды команд

`kbd` имитирует нажатие клавиши по scan-code:

```text
kbd 02
kbd 27
```

Код задается в шестнадцатеричном виде и должен быть меньше `28h`. В сценарии
клавиша нажимается сразу в ядре эмулятора.

`cmd` имитирует ввод операции МК-61 по ее opcode:

```text
cmd 10
cmd 14
```

Эта команда полезна, когда нужно выполнить ту же последовательность клавиш,
которая соответствует байту программы МК-61. Для загрузки программы в память
обычно лучше использовать `hin` или `asm`, а не `cmd`.
Допустимый диапазон opcode для `cmd` — `00`...`EF`.

## Ассемблерный ввод

`asm` записывает программу мнемониками:

```text
asm 0000 1 2 add hlt
```

После `asm` можно указать начальный адрес трансляции из четырех цифр. Если адрес
не указан, используется текущий адрес трансляции. Мнемоники можно вывести
командой `isa`.

`ins` вставляет opcode в программу и сдвигает последующие команды:

```text
ins 10 20
```

Первый аргумент - десятичный шаг программы, второй - hex-код команды.

`set$` - служебная форма записи hex-строки:

```text
set$0000 01020304
```

Для ручных `.m61` предпочтительнее `hin`, потому что синтаксис читается
однозначнее.

## Справочник команд

### Сценарные команды

| Команда | Назначение |
| --- | --- |
| `run` | Запустить текущую программу МК-61 и ждать остановки. |
| `run :label` | Перейти на метку в текущем `.m61`. |
| `run <path>` | Открыть сохраненный файл, как `open <path>`. |
| `open <path>` | Открыть или запустить сохраненный файл с возвратом в сценарий. |
| `load <номер>` | В `.m61` выполнить сценарий `M1` с именем номера. |
| `load <path.m61>` | В интерактивном терминале загрузить M61 по пути. |
| `if <условие> <команда>` | Выполнить команду при истинном условии. |
| `trap <address> run :label` | Перехватить шаг программы до выполнения. |
| `ret` | Вернуться из ловушки/вложенного файла или завершить корневой файл. |
| `print "..."` | Вывести строку, escapes и значения регистров. |

### Программная память МК-61

| Команда | Назначение |
| --- | --- |
| `hin <addr> <hex>` | Записать hex-байты в программную память. |
| `hout` | Вывести программную память hex-блоками. |
| `asm [addr] <mnemonics>` | Собрать мнемоники в программную память. |

## Команды с подтверждением

Некоторые команды опасны или интерактивны:

```text
clr
save <slot|path.m61>
sdel <slot>
sera
rst
dfu
```

Эти команды, а также `pwd`, `cd`, `ls`, `mkdir`, `mv`, `rm`, `rmdir`, `df`,
`fsclean` и `snm`, в `.m61` запрещены. Сценарий не может перезагрузить
устройство, войти в DFU или административно изменять каталог. Для загрузки
программы не нужно предварительно делать `clr`: строки `hin` записывают нужные
байты напрямую.

## Разрешения сценария

В `.m61` действует явный список разрешенных команд:

- управление потоком: `run`, `open`, `load`, `if`;
- программная память: `hin`, `set$`, `asm`, `ins`, `list`, `dump`, `pub`,
  `lasm`, `isa`, `hout`;
- калькулятор: `poke`, `R<r>=`, `reg`, `stk`, `1302`, `ring`, `kbd`, `cmd`;
- безопасный вывод и сигналы: `ver`, `led`, `beep`.

Любая новая команда терминала по умолчанию запрещена для сценария, пока ее не
добавят в этот список в прошивке. Это предотвращает случайное расширение прав
файлов `.m61` при развитии терминала.

## Ошибки выполнения

Сценарий останавливается, если:

- команда неизвестна;
- строка слишком длинная;
- аргументы команды разобрать нельзя;
- `open` или вложенный `load` не нашли файл;
- вложенность `.m61` превысила 8 уровней;
- метка некорректна, повторяется или превышен лимит в 64 метки;
- ловушка некорректна, повторяется, ссылается на неизвестную метку либо не
  помещается в активную программную память;
- обработчик ловушки дошёл до конца файла без `ret`;
- команда, запрещенная или невозможная в сценарном режиме, вернула ошибку.

После ошибки выполнение оставшихся строк не продолжается. Терминал сообщает имя
сценария, номер строки и причину; последние сведения об ошибке также сохраняются
в раннере до следующего запуска или явной отмены.


```text
:loop

```

```text
run
if x<0 open NEGATIVE.txt
if x==0 open ZERO.txt

```

```text

run

```
